<a name="what"></a>
## Что это

**QRMenu** — платформа цифрового меню для ресторанов. Гость сканирует QR-код и
открывает меню в браузере; персонал управляет содержимым через админпанель
(React-приложение «QRMenu Admin»). Эта документация описывает **полный поток
управления меню** в админке и то, как изменения отражаются на гостевой странице.

<a name="surfaces"></a>
## Две поверхности

> {primary} Любое изменение в админке отражается на **публичном гостевом меню**. Документация показывает обе стороны.

- **Админка** — редактор меню: секции, блюда, цены, модификаторы, переводы.
- **Гостевое меню** — серверная публичная страница по адресу `/{ресторан}` (и
  `/{ресторан}/t/{стол}` для заказа), которую видит гость.

<a name="flow"></a>
## Поток управления меню

Управление меню идёт сверху вниз:

1. **Меню** создаётся для ресторана (одно меню на ресторан).
2. **Секции** (категории) группируют блюда.
3. **Блюда** добавляются в секции: имя, описание, цена, фото, флаги видимости.
4. **Модификаторы** (размеры, добавки) создаются в общей библиотеке и
   привязываются к блюдам.
5. **Переопределения** меняют поведение группы модификаторов для конкретного
   блюда.
6. **Локализация** — переводы на другие языки.
7. **Гостевое меню** — предпросмотр и приём заказов.

Каждый шаг описан в соответствующем разделе слева.

<a name="quick-start"></a>
## Быстрый старт

> {info} Чтобы открыть редактор, выберите ресторан и перейдите в раздел **Меню**.

1. Откройте редактор меню — вы увидите вкладки **Блюда** и **Модификаторы**.
2. Нажмите **Добавить категорию**, введите название, сохраните.
3. Разверните категорию и нажмите **Добавить позицию** — заполните имя и цену.
4. На вкладке **Модификаторы** создайте группу (например, «Размер») пресетом.
5. В редакторе блюда привяжите группу и при необходимости задайте
   переопределения.
6. Откройте предпросмотр (значок «глаз») — проверьте, как меню выглядит для гостя.
